## はじめに
みなさんは普段AIを使用して開発を行なっていますか?
私は、普段Claude Codeを使用して仕事とプライベートで開発を行っています。

様々なAI利用方法があると思いますが、VSCodeの拡張機能を使用したり、ターミナルを使用したりしてAIを立ち上げて使用しているかと思います。

私はこれ↓を使用しています。

![](https://storage.googleapis.com/zenn-user-upload/531c2c23776e-20260314.png)

これはClaude Codeしか起動しないターミナルを内包したエディタです。
また、1つのAIがそれぞれのAIに対して実行を行うというよりは、各ディレクトリ配下に対して別々に実行を行うような形式になっています。
中心の部分でClaudeに指示を出して、右側でプロンプトを投げるためのメモを作成したり、なんやかんややったりするところです。

tmuxなどを使用してマルチエージェントとして使用している方がいますが、私はどうしてもターミナル上での操作が慣れず、最終的にアプリを作成しました。
しかも、このクオリティがバイブコーディングのみで作成できたので、エンジニア初心者の方でも気軽に作成できるかなと思います。

今回は、このエディタの作成についてご紹介します。

## このエディタについて
このアプリはAIが開発することを前提としたエディタアプリです。
PC上で動かすことを想定しています。

### 主な技術スタック
- TypeScript : フロントエンド
- React : フロントエンドフレームワーク
- xterm.js : ターミナルエミュレータ
- Node.js : バックエンド
- Electron : デスクトップアプリケーションフレームワーク
- vite : ビルドツール
など...

### 画面レイアウト

① : サイドバー ... ディレクトリごとに操作するAIを選択
② : ターミナル ... Claude Codeのみが立ち上がる

## 何ができるのか?/何ができないのか?
### 1. 普段通りのAIを使用した開発ができます
当たり前ですが、AIに「〇〇を実装しておいて」、「このバグ修正しておいて」などとプロンプトを送って開発すると思いますが、このエディタでも問題なくできます。
最近だと、VSCodeなどのエディタで直接人がコーディングすることが少なくなったかと思います。(私はコーディングいっぱいしたいけど)
要件定義を行ったら後はいい感じにAIがコーディングしてくれると思います。
なので、人間がコーディングできる部分をこのエディタ上から消しています。

### 2. 1ウィンドウで複数のAIの実行を確認できます
複数の内容をAIにコーディングしてもらうとなった際、複数のウィンドウを切り替えたりする必要があります。
これが意外とめんどくさくなったり、そもそもどこに何の実行を行なっていたかがわからなくなったりします。
このアプリでは、ワンクリックでそれぞれの実行内容を切り替えることができます。
もちろん、切り替えを行なっても実行しているものは裏っ側で動いているので、上書きされたり停止したりする心配はありません。

### 3. キューイングができます
AIがこのタスクを完了したら次のタスクを頼みたいとなったときに、そのタスクを完了しないと指示内容がぐちゃぐちゃになります。
この機能を使用すれば、タスクが完了したら、キューを確認しに行って、キューにあるプロンプトを自動で入力するようになっています。
AskUserQuestionや実行のためのYes / Noが表示された場合は勝手にプロンプトは入らないようになっています。

### 4. プロンプトを保存しておけます
よく使うプロンプトをメモ帳やスニペットとして保管しているという方は多いと思います。
このアプリでは、あらかじめよく使うものを登録しておけば、ワンクリックで実行できます。
コピペしたりする手間を省けます。

### 5. メモを取れます
「何言ってんだ、メモなんて他のアプリでできるだろ」となりますが、地味に使えます。
各セッションごとに異なるメモをこのウィンドウ上で管理できるようになっています。
そのため、他のアプリでメモを取ったとしても、この内容はどこに対するプロンプトの内容だっけ?と迷うことが少なくなります。
全セッションを跨いだメモもあります。

### 6. ファイル変更ができません
AIにファイルを変更してもらうことが前提となっているので、基本的に人がファイルを触ることを想定していません。
そのため、どうしても細かい修正を人の手で行いたいとなった場合は、別途エディタを起動して行なってもらうようになります。

## どうやって開発したの?
要件定義などはAIと壁打ちしながら作成しましたが、コーディングに関しては完全にAIオンリーになっています。
バグが発生したり、改修したい箇所が発生したりしても、大まかな指示を出してAIに改修させています。
作っている途中でターミナル部分が上手く描画されないような不具合がありましたが、それもAIに全て解決させました。

## 最後に
今回は、自分好みのエディタを作成したので、その紹介を行いました。
AIを使用すれば、比較的簡単にこのエディタを作成できるので、コーディング初心者の方やAIを使い余している方が片手間で作成できるかと思います。
是非とも自分に合ったエディタを作成して、開発スピードを上げていきましょう。
